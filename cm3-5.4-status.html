This page summarises the current status of the cm3 5.4 release. See the release plan for details.

Features

Feature Status Comment
M3GC based on gcc-3.4.5 :-) Initial segmentation violation problems fixed by Olaf Wagner in this commit.
Backend based on gcc-3.4.5 was made default by Olaf Wagner in this commit.
New garbage collection support turned on by default. :-/ New "generational/incremental" garbage collection made default by Anthony Hosking in this commit.
Obsolete "enhanced" garbage collection removed from binary distributions by Olaf Wagner in this commit.
System call wrappers are now unused but have not been removed from CVS yet.
Current plotting package. :-/ Added to standard packages in this commit, but may need to be removed again (currently being discussed on the m3devel mailing list).
Current math package. :-) Added to standard packages in this commit.
Dynamic linking by default on platforms that rarely ship with static libs these days (e.g. Linux). :-/ Working locally, not committed yet.
Documentation about how to configure linking for Modula3 programs. :-( Not even started yet.
M3IDE (Reaktor) ?? Status unknown - Randy Coleburn said he was going to begin working on this.
m3gdb :-( I tried using it on Fedora Core 5, but it failed, probably also because of address space randomisation. Sources not yet imported into CVS.
